Algorithm Docs

Understanding the 0/1 Knapsack and Min-Heap optimization algorithms

code Problem Formulation

The Network Quality of Service (QoS) challenge is modeled as a classic 0/1 Knapsack Problem. When total incoming packet sizes exceed the available bandwidth, the router must choose which packets to drop and which to transmit.

  • Capacity (W): The total bandwidth limit currently available on the router.
  • Items (n): The incoming network packets currently in the queue.
  • Weight (w): The byte size of each network packet.
  • Value (v): The administrative priority score of the packet (e.g., Academic = 95, Download = 10).

functions Dynamic Programming Application

We construct a 2D array dp[n+1][W+1] where dp[i][w] represents the maximum priority score achievable using a subset of the first i packets with a total bandwidth limit of w.

for i = 1 to n:
    for w = 0 to W:
        if weight[i] <= w:
            dp[i][w] = max(dp[i-1][w], value[i] + dp[i-1][w - weight[i]])
        else:
            dp[i][w] = dp[i-1][w]

Time Complexity

O(n * W)

Pseudo-polynomial time. Efficient for reasonable bandwidth window sizes.

Space Complexity

O(n * W)

Required for backtracking to identify specifically which packets were selected.

data_object Min-Heap Algorithm

A Min-Heap (Priority Queue) is used to always find the least-congested router in O(log R) time. When a student connects, we heappop() the lightest router, assign the student, then heappush() it back. 

for each student:
    router = heapq.heappop(heap)     # O(log R)  --  get least loaded
    router.active_users += 1
    heapq.heappush(heap, router)     # O(log R)  --  rebalance heap

Total: O(S x log R)  |  S = students, R = routers

Time Complexity

O(S x log R)

Each of S students requires 2 heap operations on R routers.

Space Complexity

O(R)

Only the R router objects are stored in the heap.
